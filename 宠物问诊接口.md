## 统一响应格式

所有接口均返回如下 JSON 结构（流式除外）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | `0` 表示成功，`1` 表示业务错误 |
| `info` | object/null | 业务数据，失败时为 `null` |
| `tip` | string | 提示文字，成功为 `"响应成功"`，失败为错误描述 |

**成功示例:**
```json
{ "code": 0, "info": { ... }, "tip": "响应成功" }
```delete


**失败示例:**
```json
{ "code": 1, "info": null, "tip": "session_id 'abc' 不存在" }
```

## 2. 问诊模块

### 2.1 POST /session/new — 创建问诊会话

创建一个新的问诊 session，返回 `session_id`。Session 同时写入数据库。

**请求体 (JSON):**

| 参数 | 类型 | 必填 | 示例值 | 说明 |
|------|------|------|--------|------|
| `pet_id` | string | 是 | `"123"` | 宠物 ID，须为数字字符串，从 `pet.t_pet_info` 查询 |

```json
{
  "pet_id": "123"
}
```

**curl 示例:**

```bash
curl http://{host}/session/new -X POST \
  -H "Content-Type: application/json" \
  -d '{"pet_id": "123"}'
```

**响应:**

```json
{
  "code": 0,
  "info": {
    "session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512",
    "pet_id": "123",
    "pet_info": {
      "name": "帕奇",
      "breed": "英短",
      "age": "2岁",
      "weight": "4.5",
      "gender": "公"
    }
  },
  "tip": "响应成功"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `session_id` | string | 32位十六进制，后续所有请求均需携带 |
| `pet_id` | string | 实际使用的宠物 ID |
| `pet_info` | object | 宠物基础信息，字段按实际数据库记录返回，均为可选 |

**错误响应（pet_id 不存在）:**

```json
{
  "code": 1,
  "info": null,
  "tip": "pet_id '123' 不存在"
}
```

### 2.2 POST /messages — 同步问诊（非流式）

发送一条用户消息，等待完整诊断回复后返回。适合不支持 SSE 的客户端。

**请求体 (JSON):**

| 参数 | 类型 | 必填 | 示例值 | 说明 |
|------|------|------|--------|------|
| `session_id` | string | 是 | `"a3f9c2e1..."` | 由 `/session/new` 返回 |
| `text` | string | 是 | `"我家猫今天不吃东西"` | 用户描述症状的口语化输入 |

```json
{
  "session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512",
  "text": "我家猫今天不吃东西，精神也不太好"
}
```

**curl 示例:**

```bash
curl http://{host}/messages -X POST \
  -H "Content-Type: application/json" \
  -d '{"session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512", "text": "我家猫今天不吃东西"}'
```

**响应:**

```json
{
  "code": 0,
  "info": {
    "user_input": "我家猫今天不吃东西，精神也不太好",
    "consultation": "您好，猫咪出现食欲下降和精神不振，可能有以下几种原因：\n\n1. **消化系统问题**：如胃肠炎、便秘...\n2. **感染性疾病**：如上呼吸道感染、猫瘟...\n\n请问猫咪最近有无呕吐或腹泻症状？体温是否正常（正常范围38-39.5℃）？"
  },
  "tip": "响应成功"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_input` | string | 原始用户输入（原样回显） |
| `consultation` | string | AI 诊断建议/追问内容 |

### 2.3 POST /messages/stream — 流式问诊（SSE）

发送一条用户消息，以 SSE（Server-Sent Events）方式流式返回诊断内容。**推荐使用此接口**。

**请求体 (JSON):** 同 [2.2 /messages](#22-post-messages--同步问诊非流式)

**curl 示例:**

```bash
curl http://{host}/messages/stream -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{"session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512", "text": "我家猫今天不吃东西"}' \
  --no-buffer
```

**SSE 响应格式:**

响应 Content-Type 为 `text/event-stream`，每个事件格式为：

```
event: <事件名>
data: <JSON 字符串>

```

（事件之间以 `\n\n` 分隔）

**事件类型:**

| event | data 字段 | 触发时机 | 说明 |
|-------|-----------|----------|------|
| `start` | `{"user_input": "..."}` | 首帧立即发出 | 前端可立即回显用户消息 |
| `delta` | `{"text": "..."}` | 每个 LLM token chunk | 累积拼接得到完整回复 |
| `done` | `{"full_text": "..."}` | 流结束 | 完整回复文本，用于对账校验 |
| `error` | `{"message": "..."}` | 发生异常 | 200 已发出，仅作语义标记 |

**响应示例 (原始字节流):**

```
event: start
data: {"user_input": "我家猫今天不吃东西"}

event: delta
data: {"text": "您好，"}

event: delta
data: {"text": "猫咪出现食欲下降"}

event: delta
data: {"text": "，可能有以下几种原因："}

event: done
data: {"full_text": "您好，猫咪出现食欲下降，可能有以下几种原因：..."}

```

**前端处理建议:**

```typescript
const resp = await fetch('/messages/stream', { method: 'POST', ... });
const reader = resp.body.getReader();
const decoder = new TextDecoder('utf-8');
let buf = '';

while (true) {
  const { value, done } = await reader.read();
  if (done) break;
  buf += decoder.decode(value, { stream: true });
  let idx;
  while ((idx = buf.indexOf('\n\n')) !== -1) {
    const frame = buf.slice(0, idx);
    buf = buf.slice(idx + 2);
    // 解析 frame 中的 event: 和 data: 行
  }
}
```

### 2.4 POST /report — 生成问诊报告

在完成至少一轮问诊后，生成最终诊断报告。相同轮次下重复请求直接返回缓存，不重复调用 LLM。

**请求体 (JSON):**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `session_id` | string | 是 | 由 `/session/new` 返回 |

```json
{
  "session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512"
}
```

**curl 示例:**

```bash
curl http://{host}/report -X POST \
  -H "Content-Type: application/json" \
  -d '{"session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512"}'
```

**响应:**

```json
{
  "code": 0,
  "info": {
    "report": "## 问诊报告\n\n**宠物**: 帕奇（英短，2岁）\n\n### 症状摘要\n...",
    "disease_card": [
      {
        "name": "猫上呼吸道感染",
        "probability": 55,
        "risk_level": "高",
        "definition": "由疱疹病毒或杯状病毒引起的上呼吸道炎症",
        "cause": "病毒感染，常见于多猫环境或应激后免疫下降",
        "symptoms": "流鼻涕、打喷嚏、食欲下降、精神萎靡",
        "diagnosis": "血常规 + PCR 病原检测",
        "treatment": "抗病毒支持治疗，必要时辅助抗生素防止继发感染"
      },
      {
        "name": "猫传染性鼻气管炎",
        "probability": 30,
        "risk_level": "中",
        "definition": "猫疱疹病毒 1 型引起的急性呼吸道传染病",
        "cause": "直接接触感染猫或其分泌物",
        "symptoms": "结膜炎、角膜溃疡、上呼吸道症状",
        "diagnosis": "PCR 检测疱疹病毒",
        "treatment": "抗病毒眼药水 + 全身支持疗法"
      }
    ],
    "medical_solutions": "建议带猫咪前往宠物医院进行血常规检查...",
    "symptom_summary": "食欲下降、精神不振，持续约1天",
    "primary_disease": "猫上呼吸道感染"
  },
  "tip": "响应成功"

```

| 字段 | 类型 | 说明 |
|------|------|------|
| `report` | string | 整体诊断说明（150-250字） |
| `disease_card` | array | 可能疾病列表，按概率降序 |
| `disease_card[].name` | string | 疾病名称 |
| `disease_card[].probability` | int | 患病概率 0-100（整数，不带 %） |
| `disease_card[].risk_level` | string | 风险等级：`高` / `中` / `低` |
| `disease_card[].definition` | string | 疾病定义 |
| `disease_card[].cause` | string | 主要病因 |
| `disease_card[].symptoms` | string | 典型临床表现 |
| `disease_card[].diagnosis` | string | 诊断方式 |
| `disease_card[].treatment` | string | 治疗方向 |
| `medical_solutions` | string | 推荐检测流程（Markdown 列表） |
| `symptom_summary` | string | 症状摘要（30字以内） |
| `primary_disease` | string | 主要诊断（同步写入长期记忆） |

**错误响应:**

```json
{
  "code": 1,
  "info": null,
  "tip": "请至少完成一轮问诊后再生成报告"
}
```

> 以下接口从 PostgreSQL 数据库查询，无需 session 在内存中存活。

**请求体 (JSON):**

```bash
curl http://{host}/session/by-pet -X POST \
  -H "Content-Type: application/json" \
  -d '{"pet_id": "123"}'
```

**响应:**

```json
{
  "code": 0,
  "info": {
    "pet_id": "123",
    "sessions": [
      {
        "session_id": "a3f9c2e1d45b4f8e92c1a7b3d6e0f512",
        "title": "我家猫今天不吃东西",

        "created_at": "2026-05-20 09:12:05",
        "updated_at": "2026-05-20 09:15:41"

## 附录

### 典型调用流程

```
1. POST /session/new     → 获取 session_id
         ↓
2. POST /messages/stream → 流式发送第一条消息（首轮 → 追问）
         ↓
3. POST /messages/stream → 继续对话（多轮）
         ↓
4. POST /report          → 生成最终诊断报告
         ↓
5. POST /session/delete  → 清理会话（可选）
```

### 数据库表结构

所有表位于 `llm` schema 下，表名前缀 `t_`。完整 DDL 见 `scripts/init_db.sql`。
